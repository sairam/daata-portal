Daata Project

Project's Vision is to Solve repeating developer painpoints

Use Cases Solved

  • Versioned Documentation by Project for teams inside/across companies. (Google Docs just wont cut it)
  • Inspire your team by displaying Code Coverage on your Status Screen
  • Reviewing designs/HTML files which cannot be deployed Detail
  • Fetching information from database and pass around without styling HTML to business teams Detail
  • Simple dashboards from code coverage or any other data without writing code Detail
  • Wishing to have a bitly/shortners without analytics (aka shortcuts) Detail
  • The Pastebin you wish you had
  • Get data from 10+ servers and uniq to pass to other teams Detail
  • Seeing that SMS/Text/Email on the browser instead of spamming the customer on stagingDetail
  • This is That Miscellaneous server/service which the DevOps never wants to manage
  • 1-Click install on DigitalOcean
  • Simple backup and restores
  • Host Internally without restrictions

Solutions In Progress

In the Pipeline/Yet To Solve

  • Proxying requests
  • Debugging HTTP/REST calls across services on staging/development Detail
  • Proxy/Replay HTTP/REST requests during development (Coming soon.. gRPC) Detail

Who is aimed for?

The tool is aimed for small development teams who want to host data.
The vision of this service is not to solve a problem for its scale, but to help solve problems for hundreds of thousands of developers who are starting out on code bases they don't understand or cases where they dont want to spend time solving.

One Liner Purpose

Why the name Daata?

Naming is one of the biggest pain point for a developer. The name 'Daata' was kinda cooked up since we want to deal with business/teams useful Data

Any PainPoints using the name Daata?

Typing it on Mobile usually autocorrects Daata to Data (irony!)

No :)

Is this production ready?

I'd say Alpha. That means "use it at your own risk, the advertised functionalities work as intended, but may fail on unintented usecases"
